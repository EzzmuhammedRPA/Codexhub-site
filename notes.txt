ملف ملاحظات - Git و GitHub (CodeX)


1) تعريف سريع
- Git: نظام تحكم في الإصدارات موزع (DVCS).
- GitHub: منصة لاستضافة مشاريع Git وتسهيل التعاون.


2) أوامر أساسية
- git init -> تهيئة مستودع جديد
- git clone <url> -> نسخ مستودع بعيد
- git status -> حالة الملفات
- git add <file> -> إضافة ملفات للـ staging
- git commit -m "msg" -> حفظ تغييرات محليًا
- git push -> رفع التغييرات للمستودع البعيد
- git pull -> جلب ودمج التغييرات من البعيد
- git checkout -b <branch> -> إنشاء والتبديل لفرع جديد
- git merge <branch> -> دمج فرع


3) فروق مهمة
- Git = الأداة اللي تدير الإصدارات على جهازك.
- GitHub = موقع/خدمة بتستضيف مستودعات Git وبضيف طبقات تعاون.


4) أفضل الممارسات
- اعمل Commits صغيرة وواضحة.
- استخدم Branches لكل ميزة.
- اختبر قبل الدمج.
- اكتب وصف واضح للPRs.
